Classroom & Video Model Design

User Model

name String
email String, unique
passwordHash String
role Enum ['teacher', 'student']
const UserSchema = new Schema({
  name: { type: String, required: true },
  email: { type: String, required: true, unique: true }
  // additional fields
});

Classroom Model

name String
teacherId ObjectId, ref: 'User'
studentIds [ObjectId], ref: 'User'
videoIds [ObjectId], ref: 'Video'
const ClassroomSchema = new Schema({
  name: { type: String, required: true },
  teacherId: { type: Schema.Types.ObjectId, ref: 'User' }
  // additional fields
});

Video Model

title String
uploaderId ObjectId, ref: 'User'
originalPath String
streamPath String
const VideoSchema = new Schema({
  title: { type: String, required: true },
  uploaderId: { type: Schema.Types.ObjectId, ref: 'User' }
  // additional fields
});

Challenge: Schema Relations with Mongoose

Challenges

  • Document references across collections
  • No native joins (unlike SQL databases)
  • Managing array references (e.g., studentIds)

Solutions

// Populate video data in classroom

await Classroom.findById(id)
  .populate('teacherId')
  .populate({
    path: 'videoIds',
    select: 'title uploaderId streamPath'
  })
